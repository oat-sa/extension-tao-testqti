# HKD-1620 - Outcome declaration identifier validation state

## Summary
- fix FE validation flow for outcome declaration identifier in test authoring
- always revalidate identifier on blur (no early return based on original value)
- clear stale validation markers before each validation pass
- keep duplicate check excluding currently edited outcome (by serial, fallback by original identifier)

## Root Cause
Validation state could become stale after an invalid edit and subsequent correction because blur handling skipped revalidation in some paths and uniqueness checks could include the currently edited declaration.

## Changes
- `views/js/controller/creator/views/test.js`
  - reworked identifier blur validation logic
  - removed skip-on-unchanged branch
  - added self-exclusion in uniqueness check
  - normalized boolean validation for identifier format

## Validation
- manual authoring flow verified during investigation:
  - invalid identifier with space => validation error + save disabled
  - corrected identifier => validation cleared + save re-enabled

## Risks
- low risk, scoped to client-side validation path in test authoring outcome declarations
